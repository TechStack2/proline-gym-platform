## Synthesis — the systemic root causes (Phase-2 seeds)

1. **Raw-hex escape hatches in core chrome.** The token system is real and works (light byte-identical, dark inverts, brand ramps) — every dark-mode A/B finding traces to values that bypassed it: `text-[#252525]` titles, pastel tint cards, role-color hex maps, native inputs, light-pinned chips. Phase 2 rule: *no color literal outside the token files; designed-fixed surfaces get explicit fixed tokens* (the DS-3 rule, generalized).
2. **No enforced primitive layer.** button/card/badge/input/select primitives exist but adoption is 30–60%, `ui/select` is used once, and there is no PageHeader, no Dialog, no EmptyState, no StatusChip. Every consistency finding (four tab styles, duplicated status pills, focus-ring drift, three empty-state tiers) is a missing-or-unadopted primitive.
3. **Tab-bar IA grew by accretion past its capacity.** Staff got the 4+More design; portal (7) and coach (6) just kept appending. There is no capacity rule, no overflow pattern outside staff, and the bar's ARIA is a tablist pretending. One shared, capacity-capped, properly-labeled TabBar with a role-specific IA decision is the fix — CSS cannot save seven tabs at 390.
4. **No shared formatting/bidi layer.** Dates in three formats, mixed clocks, currency symbol sides, phone `+` flips, raw enums, raw i18n keys, `toLocaleString` dumps — every value that meets Arabic text or a user's eye is formatted ad hoc. Phase 2: one `fmt` module (date/time/money/phone/enum-label) with mandatory bidi isolation, plus a missing-key CI gate.
5. **The PWA chrome layer is authored but not engaged.** Safe-area padding exists with no `viewport-fit=cover`; the per-tenant manifest exists but hardcodes start_url/lang/dir/splash/description; Apple install metadata is absent; the install promo speaks the wrong tenant, role, device and language. "Installable" shipped; "installed-grade" did not.
6. **Two desktop strategies.** Staff desktop was designed (sidebar, search header, real grids); portal/coach desktop is the mobile shell with a rail — header under the rail, phone column in a void, wrong-side RTL margin. Phase 2 must decide: either portal/coach desktop is a first-class layout (recommended: they're the customer-facing shells) or it's explicitly a centered mobile frame — the half-state is the worst of both.
7. **No zero/empty-state doctrine.** Alarm-colored $0s with live CTAs, "Inbox zero" above pending work, marketing copy interpolating 0, placeholder walls on public pages, dead zero-count filters — versus the exemplary Notifications empty state. One doctrine: zeros are calm, summaries must agree with their page, public surfaces collapse rather than placeholder.
8. **Brand and semantic color are one ramp.** Crimson is simultaneously brand, action, error, category and logout; the landing doesn't consume the tenant ramp it emits; portal/coach chrome shows no tenant identity. Phase 2: split roles (brand / action / destructive / status / category-neutral) and finish the WL story on the landing + install layer.
